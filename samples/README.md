# ClawCam Samples

This directory contains deterministic Phase 1 sample payloads for gateway, schema, and brain-tool development before real camera hardware is connected.

## Directories

| Directory | Purpose |
|---|---|
| `payloads/` | Hand-authored example device, event, and health payloads. |
| `node-simulator/` | Output location for generated simulator bundles. |
| `media/` | Placeholder location for sample media references. |
| `firmware-bundle/` | SD-card-style firmware artifact fixture with `events/`, `media/`, and `metadata/`. |

## Generate Simulator Payloads

```bash
cd gateway
PYTHONPATH=. python -m clawcam_gateway.simulator.cli --output ../samples/node-simulator
```

## Import Payloads

The gateway import CLI can load sample payloads into a local SQLite database:

```bash
cd gateway
PYTHONPATH=. python -m clawcam_gateway.ingest.cli import-sample ../samples/payloads --db ../clawcam_gateway.db
```

It can also import firmware SD-card bundles generated by the ESP32-S3-EYE bench firmware path:

```bash
cd gateway
PYTHONPATH=. python -m clawcam_gateway.ingest.cli import-firmware-bundle \
  ../samples/firmware-bundle \
  --db ../clawcam_gateway.db \
  --media-dir ../gateway_media
```

The firmware importer validates `events/*.json`, auto-registers unknown firmware nodes by default, copies or references media, and stores events in the same gateway database used by the dashboard and agent tools.
